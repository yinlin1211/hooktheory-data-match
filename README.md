# Hooktheory 样本匹配

将本地音乐文件与 YouTube video_id 匹配，通过 `youtube_to_keys` 生成 Hooktheory 样本 ID。

---

## 匹配算法

本地文件名由 YouTube 标题直接保存，存在以下系统性差异：

| 差异类型 | 示例 |
|---|---|
| `?` 被替换 | `How Bad Can I Be?` → `How Bad Can I Be_` |
| `/` 被替换 | `AC/DC` → `AC_DC` |
| 末尾重复标记 | `Song Title (1).mp3` |
| 标点差异（引号、冒号）| `"Title"` → `'Title'` |
| emoji 国旗被替换 | `🇦🇹 Austria` → `AT Austria` |

算法按优先级依次尝试 13 种规范化策略 + 前缀匹配，命中即停止：

| 策略 | 命中数 |
|---|---|
| 精确匹配 | 10,123 |
| 全标点忽略 | 1,211 |
| `?→_` / `→_` | 286 |
| 去末尾 `(n)` | 184 |
| 去 `(n)` + 全标点忽略 | 27 |
| 前缀匹配 | 1 |
| **合计** | **11,766** |

经逐条核查，**误匹配数为 0**。

## 复现

```bash
# 准备数据文件（放在同目录）：
#   11892.json、总json.json、youtube_to_keys.json

python3 match_final.py
# 输出：local2youtubeid.json、unmatched_local_final.json
```
