# Hooktheory 样本匹配

将本地音乐文件与 YouTube video_id 匹配，通过 `youtube_to_keys` 生成 Hooktheory 样本 ID。

---

## 数据概览

### 原始规模（youtube_to_keys）

| 指标 | 数值 |
|---|---|
| 总 video_id 数 | 15,006 |
| 总 key 数（样本上限）| **25,861** |

### YouTube 视频访问情况

| 状态 | 视频数 | 对应 key 数 |
|---|---|---|
| 访问成功（获得标题）| 12,463 | 21,488 |
| 无标题（多次访问无内容，可能已下架）| 2,542 | 4,373 |

### 本地文件与匹配情况

| 指标 | 数值 |
|---|---|
| 本地实际下载文件数 | **11,892** |
| 成功匹配到 video_id | 11,766 |
| 匹配到的唯一 video_id 数 | 11,746 |
| 有标题但未下载（版权 / 下架 / 私密）| 717 视频，1,374 个 key |
| 无法匹配的本地文件 | 60 |

### 最终样本结果

| 指标 | 数值 |
|---|---|
| 最终生成的 key 数 | **20,113** |
| files20072 收录的 key 数 | 20,072 |

**损失路径：25,861 → 21,488（无标题视频）→ 20,114（有标题但未下载）→ 20,113（最终）**

## 文件说明

### 输入文件

| 文件 | 说明 |
|---|---|
| `本地文件11892.json` | 本地音乐文件名列表（11,892 个 .mp3 文件名）|
| `视频访问情况.json` | YouTube URL → 标题映射，含访问失败记录（15,005 条）|
| `youtube_to_keys.json` | video_id → 样本 key 列表（15,006 个视频，25,861 个 key）|

### 输出文件

| 文件 | 生成依赖 | 说明 |
|---|---|---|
| `local2youtubeid.json` | `本地文件11892.json` + `视频访问情况.json` | 本地文件名 → video_id（11,766 条）|
| `unmatched_local_final.json` | 同上 | 无法匹配的本地文件（60 条）|

### 算法脚本

| 文件 | 说明 |
|---|---|
| `match_final.py` | 多策略匹配算法，读取输入文件，生成输出文件 |

## 匹配算法

本地文件名由 YouTube 标题直接保存，存在以下系统性差异：

| 差异类型 | 示例 |
|---|---|
| `?` 被替换 | `How Bad Can I Be?` → `How Bad Can I Be_` |
| `/` 被替换 | `AC/DC` → `AC_DC` |
| 末尾重复标记 | `Song Title (1).mp3` |
| 标点差异（引号、冒号）| `"Title"` → `'Title'` |
| emoji 国旗被替换 | `🇦🇹 Austria` → `AT Austria` |

算法按优先级依次尝试 13 种规范化策略 + 前缀匹配，命中即停止：

| 策略 | 命中数 |
|---|---|
| 精确匹配 | 10,123 |
| 全标点忽略 | 1,211 |
| `?→_` / `→_` | 286 |
| 去末尾 `(n)` | 184 |
| 去 `(n)` + 全标点忽略 | 27 |
| 前缀匹配 | 1 |
| **合计** | **11,766** |

经逐条核查，**误匹配数为 0**。

---

## 复现

将以下文件放在同一目录下运行脚本：

```
本地文件11892.json
视频访问情况.json
youtube_to_keys.json
files20072.json        （可选，用于验证结果）
```

```bash
python3 match_final.py
# 输出：local2youtubeid.json、unmatched_local_final.json
```
